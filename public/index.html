Event Outcome Knowledge Base
What it is: a database of what each story-event choice actually gives (stat gains, energy, skill hints, conditions). A known outcome means the bot has the effects for that event's choices and can compare them.

How it fills up: Icarus is an API bot, so it already sees the stat changes before/after every choice it makes — it auto-records outcomes from your own careers (no Frida or external dumper needed). You can also import a static outcomes.json below to seed events you haven't run yet; observed data from your runs takes precedence.

How it's used: in Event Choices, the bot scores each option's known outcome against your event stat priority and picks the best; the data also enriches the AI Dataset and gives the Local LLM better context. Events with no data yet show "effect not in database" until they're observed or imported.

MULTI-ACCOUNT CONTROL

Multi-Account Control runs several game accounts at once — each in its own bot instance, on its own port. The Manager launches and supervises one dashboard per account and auto-restarts any that crash or stall. Each account keeps its own saved login, so they never share or overwrite each other's account.

How to run two (or more) accounts
  1. Add one row per game account. Click ADD ACCOUNT for each. Give every account a unique Name and a unique Port — e.g. main / 1616 and alt / 1617.
  2. Save the roster. Click SAVE ACCOUNTS.
  3. Launch the Manager. Click LAUNCH MANAGER. It starts one bot per account on its port and keeps them alive.
  4. Log into each port separately. Open each account's dashboard (e.g. http://127.0.0.1:1616 and http://127.0.0.1:1617) and sign in with that account's Steam credentials. Every port remembers its own login.
  5. Start a career on each dashboard as usual — they run independently. Watch live status for all accounts in the list below.
Different ports alone are not enough. Each account also needs a different Name — the name is what isolates that account's saved login. Same name = same login (even on a different port); the port just decides which dashboard you open. Both Name and Port must be unique per account.

Userdata folder

Your settings, presets, accounts, and Steam auth tokens live in a userdata folder. Pointing this somewhere stable means your data survives version upgrades — even when you download a new build into a completely separate folder instead of overwriting.

How resolution works (advanced)
  1. $ICARUS_USERDATA_DIR environment variable
  2. $SWEEPYCL_USERDATA_DIR / $SWEEPYCLAUDE_USERDATA_DIR (legacy env vars)
  3. The pointer file at — set by this dialog and survives across version installs
  4. Icarus_userdata sibling folder next to the build
  5. SweepyCL_userdata / SweepyClaude_userdata (legacy sibling folders)
  6. Inside the build folder itself (not recommended — will be lost on upgrade)
Tip — the folder doesn't have to exist yet; it'll be created. A good convention is a folder named Icarus_userdata next to your build folder.

You can reopen this dialog any time from the USERDATA button in the top bar — that's also where to change your folder later if you tick "Do not show again".
